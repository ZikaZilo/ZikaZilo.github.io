Kotlin Basic Introduce

An introduction to Kotlin's basic grammar for variables and functions

Jan 01, 2025
3 minute read

Kotlin is a statically typed programming language that runs on the JVM and can also be compiled to JavaScript or native code. Here is an introduction to Kotlin’s basic grammar for variables and functions:

variable

Variable Declaration

In Kotlin, variables can be declared using either val or var.

  • val is used to declare a read-only variable (similar to final in Java). Once a value is assigned to a val variable, it cannot be changed.
  • var is used to declare a mutable variable. The value of a var variable can be changed.

Syntax

  1. Immutable Variable (val):

    1

    val variableName: Type = value

  2. Mutable Variable (var):

    1

    var variableName: Type = value

Examples

  1. Immutable Variable:

    1
2

    val name: String = "John"
val age: Int = 30

  2. Mutable Variable:

    1
2

    var count: Int = 10
count = 20  // This is allowed

Type Inference

Kotlin can infer the type of the variable from the assigned value, so specifying the type is optional.

  1. Immutable Variable with Type Inference:

    1
2

    val name = "John"  // Type is inferred as String
val age = 30       // Type is inferred as Int

  2. Mutable Variable with Type Inference:

    1
2

    var count = 10     // Type is inferred as Int
count = 20         // This is allowed

Nullable Variables

In Kotlin, you can declare a variable that can hold a null value by using the nullable type syntax (Type?).

  1. Nullable Immutable Variable:

    1

    val name: String? = null

  2. Nullable Mutable Variable:

    1
2

    var age: Int? = null
age = 30  // This is allowed

Late Initialization

For properties that cannot be initialized at the time of declaration, you can use lateinit for var or lazy for val.

  1. Lateinit Variable:

    1

    lateinit var name: String

  2. Lazy Initialization:

    1
2
3

    val name: String by lazy {
    "John"
}

Function

In Kotlin, functions are declared using the fun keyword. Here is an overview of function declaration and calling in Kotlin:

Function Declaration

  1. Basic Function:

    1
2
3

    fun functionName(): ReturnType {
    // function body
}

  2. Function with Parameters:

    1
2
3

    fun functionName(param1: Type1, param2: Type2): ReturnType {
    // function body
}

  3. Function with Default Parameters:

    1
2
3

    fun functionName(param1: Type1 = defaultValue1, param2: Type2 = defaultValue2): ReturnType {
    // function body
}

  4. Single-Expression Function:

    1

    fun functionName(param1: Type1, param2: Type2): ReturnType = expression

  5. Unit-Returning Function:

    1
2
3

    fun functionName(param1: Type1, param2: Type2): Unit {
    // function body
}

Function Call

  1. Calling a Function:

    1

    functionName()

  2. Calling a Function with Parameters:

    1

    functionName(arg1, arg2)

  3. Calling a Function with Named Arguments:

    1

    functionName(param1 = arg1, param2 = arg2)

  4. Calling a Function with Default Parameters:

    1

    functionName(param1 = arg1)  // param2 uses default value

Example

// Basic function
fun greet(name: String): String {
    return "Hello, $name!"
}

// Function with parameters and default parameter
fun add(a: Int, b: Int = 10): Int {
    return a + b
}

// Single-expression function
fun multiply(a: Int, b: Int): Int = a * b
